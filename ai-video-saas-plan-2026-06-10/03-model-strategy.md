# 模型策略

## 推荐组合

主流程：

```text
GPT-5.5
TTS
RunPod InfiniteTalk
RunPod WAN 2.6 I2V / T2V
FFmpeg / Remotion
1080p 超分 + 30fps 插帧
```

高级插件：

```text
GPT Image 2 / 其他图像模型
Seedance
Kling / Veo / Runway 类高级视频模型
```

## GPT-5.5 的角色

GPT-5.5 是内容大脑，不是视频生成引擎。

负责：

- 商品卖点理解。
- 店铺介绍文案。
- 小博主脚本。
- 分镜。
- 口播台词。
- 视频标题。
- 封面文案。
- 平台风格适配。
- 合规检查。
- 提示词生成。

它可以显著提升产品体验，但不能替代：

- TTS。
- 数字人。
- AI 视频生成。
- 后期合成。
- 云 GPU。

## WAN 2.6 I2V / T2V

定位：

```text
普通高级镜头引擎。
```

适合：

- 商品图变动态镜头。
- 店铺环境氛围镜头。
- 背景 B-roll。
- 开场镜头。
- 转场镜头。
- 5-15 秒短片段。

不适合：

- 全片 5 分钟纯生成。
- 严格文字内容。
- 价格、二维码、证书直接生成。
- 长口播嘴型同步。

商业策略：

```text
按高级镜头秒数扣点。
普通套餐可送少量。
高频使用必须加购。
```

## InfiniteTalk / 数字人口播

定位：

```text
单人口播 MVP 引擎。
```

适合：

- 老板介绍店铺。
- 数字人介绍商品。
- 小博主口播。
- 用户录音驱动画面。

注意：

- 要实测单次音频时长上限。
- 长视频可能需要 15-30 秒切段。
- 需要检查嘴型、眨眼、脸部稳定性。
- 真人肖像和声音必须授权。

商业策略：

```text
按分钟限额。
超额按分钟收费。
不要无限送。
```

## Seedance

定位：

```text
高质感 AI 镜头插件。
```

特点：

- 画面质感强。
- 镜头运动更吸引人。
- 适合广告感 B-roll。
- 成本高。
- 失败重抽贵。

推荐用法：

```text
不要让整条普通视频都由 Seedance 生成。
Seedance 只负责 5-15 秒高级镜头。
按高级点数扣。
```

适合镜头：

- 奶茶杯在冰块和水果中旋转。
- 服装产品场景化展示。
- 店铺门头电影感开场。
- 商品在使用场景中动态展示。

## GPT Image 2 / 故事板

定位：

```text
关键帧和故事板生成器。
```

适合：

- 每个镜头生成起始图。
- 生成封面。
- 商品场景图。
- I2V 前的画面控制。

特点：

- 单图成本低于视频生成。
- 能提高 I2V 可控性。
- 不是成本大头。

## JoyAI-Echo

前期研究结论：

- Hugging Face 仓库模型权重大约 46GB。
- 还需要 google/gemma-3-12b-it。
- 官方代码在 jd-opensource/JoyAI-Echo。
- 默认生成规格约 1280x736、241 帧、25fps。
- 峰值显存大约 46-50GB。
- 当前主要是 T2V / 多镜头，不是 I2V。
- 许可偏研究/学术，不能直接作为商业核心。

结论：

```text
JoyAI-Echo 不适合作为这个 SaaS 的商业主引擎。
可以借鉴多镜头记忆、故事板连续性、外部记忆 bank 的思路。
```

## 最佳模型分工

```text
低成本主体：
GPT-5.5 + TTS + FFmpeg / Remotion

数字人口播：
InfiniteTalk / HeyGen 类 / 其他数字人方案

普通 AI 镜头：
WAN 2.6

高级 AI 镜头：
Seedance / Kling / Veo

最终规格：
1080p30 由超分、插帧、转码保证
```
